## Contributing

When adding new UI documentation, please:
1. Follow the existing structure and format
2. Include code examples where applicable
3. Add screenshots or diagrams for visual concepts
4. Keep documentation up-to-date with code changes

## Important: Documentation Maintenance

**⚠️ Critical Requirement**: When making changes to patterns, design system, components, or layouts, you **MUST** update the corresponding documentation files. This is enforced via `.cursorrules` in the project root.

### Documentation Update Checklist
- [ ] Updated relevant documentation files
- [ ] Added code examples if introducing new patterns
- [ ] Updated examples if patterns changed
- [ ] Verified documentation accuracy

See `.cursorrules` in the project root for complete guidelines.
